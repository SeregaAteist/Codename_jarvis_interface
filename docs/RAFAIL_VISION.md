# РАФАИЛ — Концепция универсального агента корпоративных знаний
## Версия: 1.0 | Дата: 13.06.2026

---

## СУТЬ

Рафаил — не база знаний и не генератор контента.
Рафаил — **живой корпоративный мозг** который создаёт, поддерживает и развивает
систему знаний компании автономно.

Одна команда ночью:
> "Рафаил, мы берём направление производства семечек из переработанного пластика
> и нанимаем баристу для кофе в офис"

Утром готово:
- Раздел "Производство" в базе знаний
- Должностная инструкция баристы
- Курс по кофемашине (взял из мануала по модели)
- Регламент обслуживания (сервисные интервалы из документации)
- Связи с существующими разделами (закупки, логистика)
- Мониторинг цен на арабику подключён

## ПРИНЦИПЫ

**1. Минимум вопросов — максимум действий**
Рафаил уточняет только критичное. Остальное определяет сам.
Не "а коров или быков?" — а делает под оба варианта если не сказано.

**2. Без пометок "гипотетическое"**
Скрипт есть скрипт. Источник неважен пользователю.
Рафаил тихо заменяет на лучшее по мере накопления данных.

**3. Живая база — не архив**
Устаревшее помечается и заменяется автоматически.
Новое оборудование в прайсе → автоматически в курсы.

**4. Мультитенантность**
Профиль компании меняется одной командой.
Новая компания — новый профиль, новая база, тот же Рафаил.

**5. Связи важнее разделов**
Курс баристы связан с регламентом закупок кофе.
Курс монтажника связан с мануалом конкретного инвертора.
Скрипт продаж связан с реальными звонками из Ringostat.

## ИСТОЧНИКИ ЗНАНИЙ

### Документы компании
- КП (коммерческие предложения) → типовые конфигурации, кейсы
- Прайсы закупщиков (Excel/PDF) → реестр оборудования, цены
- Мануалы оборудования (PDF) → технические данные, сервисные интервалы
- Регламенты существующие → база для расширения
- Договоры, акты → юридические шаблоны

### Внешние источники
- Сайты производителей → актуальные мануалы по артикулу
- Отраслевые новости → тренды, изменения законодательства
- YouTube → видео-инструкции, обзоры оборудования
- Научные/профессиональные источники → методики, стандарты

### Операционные данные
- Ringostat → транскрипции звонков → скрипты продаж
- Kommo → сделки, возражения, конверсия → эффективность скриптов
- Moodle → прогресс сотрудников → пробелы в знаниях

## РАБОТА СО СКРИПТАМИ И ВОЗРАЖЕНИЯМИ

### Три источника (невидимы пользователю)

**Из курсов и тренингов** — стартовая база:
- Фильтруется под специфику рынка (Украина, СЭС, ценовой сегмент)
- 3-5 вариантов на возражение, не 100
- Адаптируется под сегмент клиента

**Из реальных звонков** — накапливается:
- Удачный → добавляется в скрипт
- Неудачный → помечается, ищется замена
- Контекст: тип клиента, бюджет, конфигурация

**Адаптация под сегмент:**
```
Частный дом (50-80к грн):
  "Дорого" → окупаемость 5-7 лет + зелёный тариф

Бизнес (500к+ грн):
  "Дорого" → ROI расчёт + налоговые льготы

Агро (1М+ грн):
  "Дорого" → кейсы аналогичных хозяйств + экономия на дизеле
```

### Эволюция скрипта
```
Старт:     3-5 вариантов из тренингов/курсов
Месяц 1:   +реальные удачные из звонков
Месяц 3:   неэффективные заменены
Месяц 6:   скрипт полностью на данных LK Energy
```

## ПЛАН РЕАЛИЗАЦИИ

### Этап 1 — Стиль и качество (сейчас)
- Настроить промпт под LK Energy специфику
- Убрать нерелевантный контент
- Тестировать на 3-5 секциях до нужного качества

### Этап 2 — Реструктуризация Moodle
- Пересмотреть существующую структуру
- Создать архитектуру под карьерную матрицу

### Этап 3 — Реестр оборудования
- `equipment_registry.py`
- Загрузить первые мануалы (DEA, Pylontech, JA Solar)
- `doc_parser.py` — читать PDF мануалы

### Этап 4 — Прайсы и КП
- `price_parser.py`
- `kp_parser.py`
- Автоматическое обновление реестра

### Этап 5 — Петля Ringostat
- `call_analyzer.py`
- Интеграция транскрипций в скрипты
- Автоматическое обновление возражений

### Этап 6 — Мультитенантность
- `profile_manager.py`
- Горячая замена профилей
- Новые коннекторы (Notion, Drive)

## МЕТРИКИ УСПЕХА

- Новое направление разворачивается за < 30 минут
- Скрипт обновляется автоматически после 10 звонков
- Устаревший материал заменяется без участия человека
- Новый профиль компании — одна команда в TG
- Прайс загружен → реестр обновлён → курсы дополнены автоматически
